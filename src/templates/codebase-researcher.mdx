**Purpose:** Exact pattern matching for verification and counts.

**Two-Phase Approach:**

**Phase 1: DISCOVER (no MatchPerLine)**
```
grep_search({
  Query: "TODO|FIXME|HACK",
  SearchPath: "/project/src",
  Includes: ["*.ts", "*.tsx"]
})
```
→ Returns file list only. Fast, minimal output.

**Phase 2: CONTEXT (with MatchPerLine)**
```
grep_search({
  Query: "TODO",
  SearchPath: "/project/src/specific-file.ts",
  MatchPerLine: true
})
```
→ Returns matches with surrounding code.

**Why two phases?** MatchPerLine on broad search = context explosion.
